# Persistence — Hive models, typeIds, SettingsModel fields, write timing

> Deep-dive for persistence (the Hive domain model, load-bearing typeIds, every `SettingsModel` field, and when each feature writes). Loaded on demand — see the routing table in CLAUDE.md. For "where is X" lookups use docs/CODEMAP.md.

Persistence is `hive_ce` + `hive_ce_flutter` (the community Hive fork — binary- and API-compatible with classic Hive; on analyzer 7+ so it doesn't cap the lint toolchain). Adapters are generated by `hive_ce_generator` (`@HiveType` + `part '*.g.dart'`).

## Domain model (Hive typeIds are load-bearing)

| typeId | Model | Box name | Notes |
|---|---|---|---|
| 0 | `SettingsModel` | `settings` | Single key `'current'`; loaded synchronously in `main()` |
| 1 | `HttpRequestConfig` | `history` | Shared between history and collection nodes |
| 2 | `HttpRequestTabModel` | `tabs` | Tab state including response cache; `List<StoredResponseModel> responseHistory` at `HiveField(9)` — time-travel history (next free: 10) |
| 3 | `CollectionNode` | `collections` | Nested (children list stored as `HiveField(3)`); `String? description` at `HiveField(6)`; `List<SavedExampleModel> examples` at `HiveField(7)`; `Map<String,String> variables` at `HiveField(8)`; `List<String> secretKeys` at `HiveField(9)` (next free: 10) |
| 4 | `EnvironmentModel` | `environments` | Flat list; `variables` is `Map<String, String>` at `HiveField(2)`; `List<String> secretKeys` at `HiveField(3)` (next free: 4) |
| 5 | `MultipartFieldModel` | (embedded) | Multipart/form-data field; nested inside `HttpRequestConfig` |
| 6 | `StoredCookieModel` | `cookies` | Cookie jar; keyed by `domain\|path\|name` (one put/delete per cookie); `bool hostOnly` at `HiveField(7)` (default `false` — RFC 6265 host-only flag; absent Domain attribute → exact-host match; next free: 8) |
| 7 | `ExtractionRuleModel` | (embedded) | One extraction rule; nested inside `RequestRulesModel` |
| 8 | `AssertionModel` | (embedded) | One assertion; nested inside `RequestRulesModel` |
| 9 | `RequestRulesModel` | `requestRules` | Per-request-config rules (assertions + extractions), keyed by configId |
| 10 | `SavedExampleModel` | (embedded) | A saved request+response example; nested in `CollectionNode.examples`. `name` + epoch-millis `capturedAt` + a reused `HttpRequestConfig` (carries the response columns) |
| 11 | `StoredResponseModel` | (embedded) | One captured response in a tab's time-travel history; nested as a list in `HttpRequestTabModel` (field 9). `id` + status/body/headers/`durationMs` + epoch-millis `capturedAt` |
| 12 | `PanelModel` | `panels` | Virtual-desktop workspace structure: `{id, name, List<String> orderedTabIds, activeTabId}` at `HiveField` 0–3. Stores only **ids** — tab entities stay in the `tabs` box (typeId 2). Panel order + active panel live in the `tabs_meta` box under keys `panelOrder` / `activePanelId` (next free: 4) |

**Never renumber an existing `typeId`.** Add new models with a fresh ID (**next free: 13**).

## SettingsModel fields (typeId 0)

`SettingsModel` is a single record (box `settings`, key `'current'`, loaded synchronously in `main()`). Its `@HiveField` map:

- **`themeId`** at `HiveField(7)` (default `kClassicThemeId` — `'classic'` — in both the model and the entity). Drives which theme builder is active.
- **`activeEnvironmentId`** at `HiveField(8)` (nullable, no default — `null` means "No Environment"). The id of the environment whose variables substitute into `{{var}}` placeholders at send time. `SettingsEntity.copyWith` uses a sentinel (`_unchanged` `Object` constant) so the caller can explicitly clear the id back to `null`.
- **`alwaysPrettifyLargeResponses`** at `HiveField(17)` (default `false`). When `true`, `_ResponseBodyView` prettifies + highlights bodies over `kLargeResponseViewerChars` automatically instead of the plain-text large viewer — the user accepts the render cost. The `kResponseBodyTooLargePlaceholder` sentinel always stays plain text.
- **`maxRedirects`** at `HiveField(18)` (default `5`) — see docs/architecture/network-and-cookies.md for the manual redirect loop that enforces it.
- **`clientCertPath` / `clientKeyPath` / `clientCertPassphrase`** at `HiveField(19/20/21)` (nullable; mTLS). Plain-string data — never a `dart:io` `SecurityContext`.
- **`responseHistoryLimit`** at `HiveField(23)` (default `5`; `0` disables time-travel, clamped `0..50`).
- **`saveLargeResponsesInHistory`** at `HiveField(24)` (default `true`; when `false`, *superseded* history entries over `kLargeResponseViewerChars` are downgraded to metadata-only). Both response-history fields are carried on the `SendRequest` event (computed at dispatch from `SettingsBloc`, like `envVars`) — the bloc holds no settings reference. The downgrade happens in `TabsBloc._recordResponse` (newest entry always keeps its full body); the unconditional 1 MiB `kMaxPersistedResponseBodyChars` cap is applied separately in `tabs_repository_impl._toPersistableModel`.
- **`checkForUpdatesOnStartup`** at `HiveField(25)` (default `true`; toggled via `UpdateCheckForUpdatesOnStartup`).
- **`skippedUpdateVersion`** at `HiveField(26)` (nullable; set via `SetSkippedUpdateVersion` when the user picks "Skip this version"). A still-newer release prompts again because `shouldPromptForUpdate` only suppresses the exact skipped version — the stored value is never cleared.
- **`gitUserName`** at `HiveField(28)` and **`gitUserEmail`** at `HiveField(29)` — the git commit identity, used inline (`git -c user.name=…`) by the git-sync feature, never written to the user's global git config.

**Retired fields — do not reuse these indices:** `HiveField(22)` (`reduceVisualEffects`, removed) and `HiveField(27)` (`enableThemeSounds`, removed). **(Next free: 30.)**

## Entity ↔ Model boundary

Every data-layer model implements `toEntity()` / `fromEntity()` / `copyWith()`. The domain entity is the public currency; Hive models never escape the `data/` layer.

dart run build_runner build --delete-conflicting-outputs

## Write timing

| Feature | When it writes |
|---|---|
| Settings | immediately on every `Update*` |
| Collections | immediately after every mutation (saves the whole tree) |
| History | via Hive `Box.watch()`; writes on each `add/delete/clear` |
| Tabs | debounced 10 s after any change + flush on `close()` |
| Environments | immediately after every mutation (single keyed put/delete per environment; batch import saves the list) |
